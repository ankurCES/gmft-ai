# ADR-0015 ‚Äî v0.4-A.2 LLM judge for plan quality

**Status:** Accepted (v0.4-A.2)
**Date:** 2026-06-19
**Deciders:** Ankur
**Supersedes:** none
**Related:** [ADR-0014](../adr/0014-v0.4-a-supervisor-completion.md), [v0.4 plan ¬ßA.2](../../2026-06-18-gmft-ai-v0.4.md)

## Context

ADR-0009 introduced the supervisor as a strictly-silent post-hoc
consumer. ADR-0014 added Rule E (risk-escalation). v0.4-A.2 adds
the LLM judge ‚Äî a per-turn LLM call that runs in addition to the
pure-rule engine on the first high-blast-radius destructive tool
call of the turn.

The pre-implementation doubt-driven-development review surfaced
14 findings, 7 of which were confirmed correctness bugs in the
original proposal. This ADR documents the reconciled design.

## Decision

### What the judge does

A new async function `judgePlanQuality(input, model)` returns one
of two verdicts: `'sufficient'` (don't fire) or `'insufficient'`
(fire a `plan-issue` advice message). It runs at most **once per
turn**, only when:
- `event.type === 'tool-call-request'`
- `isHighBlastRadius(event.name)` ‚Äî sqlmap or nuclei
- `state.judgeRanThisTurn === false`
- `GMFT_SUPERVISOR_JUDGE === 'true'` (env var opt-in)
- `opts.judgeModel` is provided (otherwise the judge never fires)

When the verdict is `'insufficient'`, the wrapper emits ONE
additional `plan-issue` fire (severity: warn) with the LLM's
reason in the advice text. The fire is yielded in the **same
chunk** as the existing rule fires ‚Äî BEFORE `yield event` ‚Äî so
the TUI's chunk-pairing contract holds (the ‚öÝ marker is paired
with the triggering `tool-call-request`).

### High-blast-radius set

```typescript
const HIGH_BLAST_RADIUS_TOOLS = new Set(['sqlmap', 'nuclei']);
```

**Reconciled from the original proposal's `{sqlmap, nuclei_run,
nuclei_scan}`.** The actual tool names in the registry are
`sqlmap` and `nuclei` (verified at `packages/tools/src/web/sqlmap.ts:34`
and `packages/tools/src/web/nuclei.ts:50`). The original proposal's
`nuclei_run`/`nuclei_scan` would never have matched a real tool call.

Note: nuclei has `flags: []` (not destructive) while sqlmap has
`flags: ['destructive']`. The judge treats both as
"high-blast-radius" ‚Äî orthogonal to the `destructive` flag. This
is intentional: nuclei emits active probing traffic that
operationally warrants extra caution even though the registry
doesn't flag it destructive. Future maintainers: if you add a
new high-blast-radius tool, add it to this set AND consider
whether the registry's `flags` should also be updated.

### Per-tool trigger target accessor

The original proposal read `args.target` for the trigger target.
The reconciled design reads:

```typescript
function extractTarget(name: string, args: Record<string, unknown>): string {
  // sqlmap uses `args.url`, nuclei uses `args.target`. Future HBR tools
  // should follow one of these conventions; this fallback chain handles
  // both, plus `args.host` for tools that use it.
  const a = args as Record<string, unknown>;
  if (typeof a.url === 'string') return a.url;
  if (typeof a.target === 'string') return a.target;
  if (typeof a.host === 'string') return a.host;
  return '';
}
```

The prompt renders `sqlmap` against `<url>` and `nuclei` against
`<target>` correctly, instead of the original proposal's broken
`` `sqlmap` against `` `` (always-empty target).

### Timeout

The judge enforces a **10-second hard ceiling** via `Promise.race`
with a `setTimeout`, matching the postmortem pattern at
`packages/core/src/agent/supervisor-postmortem.ts:99-131`. A
provider stall cannot freeze the supervisor loop indefinitely.
On timeout, the verdict degrades to `'sufficient'` (don't fire).

### Env-var gate placement

The original proposal put the env-var check inside
`judgePlanQuality`. The reconciled design moves it into the
wrapper integration, **before** the `await`:

```typescript
if (!judgeEnabled()) {
  // Default OFF path ‚Äî don't even enter judgePlanQuality.
  yield event;
  continue;
}
```

This means the default-OFF case (the overwhelming common case)
never enters the judge function, never queues a microtask, and
never incurs the LLM call latency tax. The env-var check inside
`judgePlanQuality` is a redundant safety net, not the gate.

### Findings summary

The original proposal passed `findingsSummary: ''` (always empty).
The reconciled design derives a one-line summary from existing
state:

```typescript
const reconCount = state.ruleC.reconCallsThisTurn;
const findingsSummary = reconCount > 0
  ? `${reconCount} recon-class tool call(s) earlier this turn (whois/dig/nmap_*/etc.)`
  : 'No recon-class tools called this turn';
```

This gives the LLM real signal without needing the findings-sidecar
wiring (which is v0.4-A.3's job). The LLM's verdict is now
informed by whether any recon happened this turn ‚Äî the most
relevant signal for "is sqlmap justified right now?"

### Verdict parser

The original proposal used two substring checks:
`lower.includes('verdict: insufficient') || lower.startsWith('insufficient')`.
The reconciled design uses a single regex:

```typescript
const insufficientMatch = /verdict\s*:\s*insufficient\b/i.exec(raw);
if (insufficientMatch) {
  return { verdict: 'insufficient', reason: raw.trim() };
}
return { verdict: 'sufficient', reason: raw.trim() };
```

This handles case-insensitivity, whitespace before the verdict
word, and a word boundary (so `insufficiently` doesn't match).
Per locked decision #6 in ADR-0014: any parse that doesn't
match `'insufficient'` (including `'unclear'`, free-form
responses, empty responses, refusal messages) is treated as
`'sufficient'` (don't fire). Operator trust > catch rate.

### Error handling

Three error paths degrade to `'sufficient'` (don't fire):
- `judgeEnabled()` is false (env var unset) ‚Üí NOOP, no LLM call
- LLM call throws (rejected promise) ‚Üí `'sufficient'`, reason
  includes the error message for the audit log
- LLM call exceeds the 10-second timeout ‚Üí `'sufficient'`,
  reason mentions timeout

The judge must NEVER silently block operators. A buggy judge
that fires spurious `'insufficient'` verdicts would inject
LLM-critic advice into every high-blast-radius tool call,
degrading operator trust in the supervisor fire breadcrumb.

### `SupervisorState` extension

The state grows by one optional field:

```typescript
export type SupervisorState = {
  // ... existing fields
  judgeRanThisTurn?: boolean; // v0.4-A.2; defaults to undefined (= false)
};
```

`createInitialState` does not set the field (defaults to
`undefined`, treated as `false`). `resetForNewTurn` sets it to
`false` explicitly so per-turn resets are obvious to future
maintainers. The field is **optional** so the 27+ existing
`createInitialState()` call sites (across `supervisor-rules.test.ts`
and `supervisor-rules-rule-e.test.ts`) compile without change.

### Advisory, not enforcement

Per ADR-0009: "the supervisor cannot block tool calls, override
the chokepoint, or prompt the user." The judge is strictly
advisory. Even when the verdict is `'insufficient'`, the
high-blast-radius tool call proceeds (the chokepoint's
destructive-confirmation gate is the only enforcement). The
judge's fire is a breadcrumb in the operator's view and a
record in the audit log ‚Äî nothing more.

This is the same model as Rules A/B/C/E: silent advice injected
into the agent's history as `role: 'user'`, plus a `supervisor-fire`
event the TUI can render as a ‚öÝ marker.

## Implementation

### New file: `packages/core/src/agent/supervisor-judge.ts`

```typescript
import type { LanguageModel } from 'ai';
import { generateText } from 'ai';

export type JudgeVerdict = 'sufficient' | 'insufficient';

export interface JudgeInput {
  recentToolCalls: ReadonlyArray<{ name: string }>;
  findingsSummary: string;
  triggerTool: string;
  triggerTarget: string;
}

export interface JudgeResult {
  verdict: JudgeVerdict;
  reason: string;
}

const JUDGE_TIMEOUT_MS = 10_000;
const INSUFFICIENT_RE = /verdict\s*:\s*insufficient\b/i;

export async function judgePlanQuality(
  input: JudgeInput,
  model: LanguageModel,
): Promise<JudgeResult> {
  let timer: ReturnType<typeof setTimeout> | undefined;
  try {
    const result = await Promise.race([
      generateText({ model, prompt: PROMPT(input), maxTokens: 256 }),
      new Promise<never>((_, reject) => {
        timer = setTimeout(
          () => reject(new Error(`judge timed out after ${JUDGE_TIMEOUT_MS}ms`)),
          JUDGE_TIMEOUT_MS,
        );
      }),
    ]);
    const raw = result.text;
    if (INSUFFICIENT_RE.test(raw)) {
      return { verdict: 'insufficient', reason: raw.trim() };
    }
    return { verdict: 'sufficient', reason: raw.trim() };
  } catch (err) {
    return { verdict: 'sufficient', reason: `judge errored: ${(err as Error).message}` };
  } finally {
    if (timer) clearTimeout(timer);
  }
}

const PROMPT = (input: JudgeInput): string => `
You are auditing an offensive-security agent's tool-call sequence...

`.trim();
```

(Full implementation in the file. The prompt is omitted here for
brevity but is identical in structure to the original proposal's
prompt, with the per-tool target accessor feeding the right
target into the prompt text.)

### Wrapper integration: `packages/core/src/agent/supervisor.ts`

The wrapper integration adds:
1. `const HIGH_BLAST_RADIUS_TOOLS = new Set(['sqlmap', 'nuclei'])`
2. `function extractTarget(name, args): string` (per-tool accessor)
3. `function judgeEnabled(): boolean` (env-var gate)
4. A new block inside the per-event loop, BEFORE `yield event`:

```typescript
// v0.4-A.2 ‚Äî LLM judge for plan quality.
// Runs at most once per turn, only when:
//   - event.type === 'tool-call-request'
//   - isHighBlastRadius(event.name)
//   - judgeEnabled() (GMFT_SUPERVISOR_JUDGE=true env var)
//   - opts.judgeModel is provided
//   - state.judgeRanThisTurn !== true
// Per ADR-0015, runs BEFORE yield event so the fire pairs with the
// triggering tool-call-request in the TUI's chunk-ordering contract.
if (
  event.type === 'tool-call-request' &&
  HIGH_BLAST_RADIUS_TOOLS.has(event.name) &&
  judgeEnabled() &&
  opts.judgeModel !== undefined &&
  !state.judgeRanThisTurn
) {
  state.judgeRanThisTurn = true;
  const reconCount = state.ruleC.reconCallsThisTurn;
  const findingsSummary = reconCount > 0
    ? `${reconCount} recon-class tool call(s) earlier this turn`
    : 'No recon-class tools called this turn';
  const input: JudgeInput = {
    recentToolCalls: state.ruleA.recent.map(r => ({ name: r.name })),
    findingsSummary,
    triggerTool: event.name,
    triggerTarget: extractTarget(event.name, event.args),
  };
  const result = await judgePlanQuality(input, opts.judgeModel);
  if (result.verdict === 'insufficient') {
    const fire: PlanIssueFire = {
      kind: 'plan-issue',
      severity: 'warn',
      text: `LLM judge: insufficient pre-attack recon for ${event.name}`,
      advice: `Supervisor (LLM judge): \`${event.name}\` invoked without sufficient pre-attack recon per the LLM judge. Reason: ${result.reason}`,
      targetEventId: event.id,
    };
    state = applyFire(state, fire);
    opts.historyRef.current = [
      ...opts.historyRef.current,
      { role: 'user', content: `Supervisor: ${fire.advice}` },
    ];
    yield { type: 'supervisor-fire', fire, targetEventId: fire.targetEventId };
  }
}
```

The `opts.judgeModel` is added to `WithSupervisorOpts` as an
optional `LanguageModel`. AgentApp does NOT wire a model in
v0.4-A.2 ‚Äî that wiring is a v0.4-A.4 follow-up (CLI/slash surface).
The judge is therefore dormant in production until the operator
opts in via env var AND wires a model in their custom AgentApp
deployment.

## Tests

`packages/core/test/supervisor-judge.test.ts` ‚Äî 6 tests:

1. `judgePlanQuality` returns `'sufficient'` when env var is unset (NOOP)
2. `judgePlanQuality` returns `'insufficient'` when LLM responds with `VERDICT: insufficient ‚Äî ...`
3. `judgePlanQuality` returns `'sufficient'` when LLM responds with `unclear` (locked decision #6)
4. `judgePlanQuality` returns `'sufficient'` on LLM error (try/catch degradation)
5. **Wrapper integration:** judge fires at most once per turn (2nd high-blast-radius call does not trigger)
6. **Wrapper integration:** judge does NOT fire when `GMFT_SUPERVISOR_JUDGE` is unset

Tests 2-4 require a mock `LanguageModel`. The `ai` package exports
`MockLanguageModelV1` (or similar) ‚Äî will verify at test-writing
time. If no mock is exported, hand-rolled stub implementing the
required methods.

Test 5-6 require the wrapper integration test surface. The
existing `supervisor-integration.test.ts` provides patterns for
this.

## Alternatives considered

**A. Use the same model as the agent's primary model.** This is
the default. The judge is in the same conversation context as
the agent's tool call ‚Äî it might systematically produce "sufficient"
verdicts because LLMs tend to justify their previous outputs.
We accept this trade-off because the cost of routing to a
separate model (latency, $$$) outweighs the false-negative rate.
If a future maintainer observes systematic bias, the
`opts.judgeModel` option allows swapping to a different model.

**B. Run the judge synchronously (block the loop on LLM call).**
This is what the reconciled design does ‚Äî the `await` blocks
the per-event loop until the LLM responds. The alternative
(yield the tool-call-request, then come back with the judge's
fire) breaks chunk ordering (finding #4 in the review). Blocking
is the lesser evil ‚Äî operator sees a brief pause before the
sqlmap/nuclei tool call lands, but the ‚öÝ marker pairs correctly
with the event in the TUI.

**C. Cache LLM verdicts within a session.** A second sqlmap call
in the same session could reuse the first call's verdict. We
rejected this: each sqlmap call targets a different URL with
potentially different recon adequacy. The cache would be wrong.

## Consequences

- `packages/core/src/agent/supervisor.ts` grows by ~30 lines
  (judge block + per-tool accessor + env-var gate).
- `packages/core/src/agent/supervisor-types.ts` grows by 1
  optional field on `SupervisorState`. Existing call sites compile.
- New file `packages/core/src/agent/supervisor-judge.ts` (~80 lines).
- New test file `packages/core/test/supervisor-judge.test.ts`
  (~6 tests, ~150 lines).
- `WithSupervisorOpts.judgeModel` is a new optional field.
  Existing callers (AgentApp) compile without change.
- The judge is **dormant in production** until the operator sets
  `GMFT_SUPERVISOR_JUDGE=true` AND wires a model. This is
  intentional ‚Äî defaults to safe.
- Workspace test total: +6 new tests (863 ‚Üí 869).
