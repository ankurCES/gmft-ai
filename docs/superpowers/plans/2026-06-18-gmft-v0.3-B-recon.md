# v0.3.B — Recon Expansion Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship 13 new recon tools (7 network + 3 web + 3 wifi) on top of the v0.3.A baseline, growing the tool catalog from 15 → 28 and the test suite from 536 → ~820.

**Architecture:** One shared Docker image per category (network/web) gets a new tag with the additional binaries baked in; wifi tools are host-only (mirroring `evil_twin`'s `sudo ./fluxion.sh` pattern — fluxion is system-installed, not in any image). All new tools follow the existing `Tool<typeof Input, typeof Output>` shape from `@gmft/core` with Zod schemas, the shared `run({ argv, image, timeoutMs })` runner, the standard `findings: Finding[]` + `mode` + `fellBack` + `durationMs` output envelope, and the `category: 'recon'` or `category: 'binary'` taxonomy. Destructive/elevation flags and `typeToConfirm` are applied per the B.1 / B.3 risk profile.

**Tech Stack:** TypeScript (NodeNext), Zod, vitest, Alpine 3.20 Docker base, the existing `@gmft/core` `run()` runner, the existing `Tool` type, `Finding` from `@gmft/core`, `tmux` for wifi session wrapping, `sudo` + system binaries for wifi host-only mode.

**Context:** The parent v0.3 plan (`docs/superpowers/plans/2026-06-17-gmft-v0.3-run-polish-and-tool-surface.md`) §Phase B has two errors that this plan corrects:

1. **Per-tool Dockerfiles:** the plan says "A `Dockerfile` at `docker/<name>/Dockerfile`". The actual repo pattern is **one shared image per category** (`docker/Dockerfile.network`, `.web`, `.wifi`) with all binaries bundled in. This plan extends the shared images and bumps the tag (`gmft/network:0.3`, `gmft/web:0.3`).
2. **PR title counts:** the plan's PR title says "10 network + 3 wifi" but the section breakdown is 7 network + 3 web + 3 wifi = 13. The corrected title is "7 network + 3 web + 3 wifi".

**Test budget:** +285 (not the +300 in the parent plan — the corrected breakdown is 7 × 20 + 3 × 20 + 3 × 25 = 140 + 60 + 75 = 275, plus ~10 catalog/tool-catalog-doc tests = ~285). Total post-B: **536 + 285 ≈ 821**.

```typescript
import { z } from 'zod';
import { run } from '../shared/runner.js';
import type { Tool, ToolContext, Finding } from '@gmft/core';

export const MasscanInput = z.object({
  target: z.string().min(1),
  ports: z.string().min(1),
  rate: z.number().int().positive(),
});
export type MasscanInputT = z.infer<typeof MasscanInput>;

export const MasscanPort = z.object({
  port: z.number().int(),
  protocol: z.string(),
  ip: z.string().optional(),
});
export type MasscanPort = z.infer<typeof MasscanPort>;

export const MasscanParsed = z.object({
  openPorts: z.array(MasscanPort),
  count: z.number(),
});
export type MasscanParsedT = z.infer<typeof MasscanParsed>;

export const MasscanOutput = z.object({
  openPorts: z.array(MasscanPort),
  count: z.number(),
  findings: z.array(z.any()),
  durationMs: z.number(),
  mode: z.enum(['host', 'host+landlock', 'host+seccomp', 'host+landlock+seccomp', 'docker']),
  fellBack: z.boolean(),
});
export type MasscanOutputT = z.infer<typeof MasscanOutput>;

const PORT_LINE = /^Discovered open port (\d+)\/(\w+) on (\S+)/;

export function parseMasscanOutput(stdout: string): MasscanParsedT {
  const openPorts: MasscanPort[] = [];
  for (const line of stdout.split('\n')) {
    const m = PORT_LINE.exec(line);
    if (!m) continue;
    openPorts.push({
      port: Number(m[1]),
      protocol: m[2]!,
      ip: m[3],
    });
  }
  return { openPorts, count: openPorts.length };
}

export function masscanFindings(parsed: MasscanParsedT, target: string): Finding[] {
  const now = Date.now();
  const slug = target.replace(/[^a-zA-Z0-9.-]/g, '-');
  return parsed.openPorts.map((p) => ({
    id: `masscan-${slug}-${p.port}-${p.protocol}-${now}`,
    tool: 'masscan',
    target,
    title: `Port ${p.port}/${p.protocol} open (masscan)`,
    description: `masscan reports ${p.port}/${p.protocol} open${p.ip ? ` on ${p.ip}` : ''}`,
    evidence: p.ip ?? '',
    severity: 'medium' as const,
    ts: now,
  }));
}

export const masscanTool: Tool<typeof MasscanInput, typeof MasscanOutput> = {
  name: 'masscan',
  category: 'recon',
  flags: ['targetRequired'],
  targetsFromFile: true,
  description:
    'High-speed TCP/UDP port scanner. Use when nmap is too slow; masscan is faster but less accurate.',
  input: MasscanInput,
  output: MasscanOutput,
  async run(input: MasscanInputT, _ctx: ToolContext): Promise<MasscanOutputT> {
    const parsed = MasscanInput.parse(input);
    const argv = [
      'masscan',
      parsed.target,
      `-p${parsed.ports}`,
      `--rate=${parsed.rate}`,
    ];
    const r = await run({ argv, image: 'gmft/network:0.3', timeoutMs: 120_000 });
    const parsed2 = parseMasscanOutput(r.stdout);
    const findings = masscanFindings(parsed2, parsed.target);
    return {
      openPorts: parsed2.openPorts,
      count: parsed2.count,
      findings,
      durationMs: r.durationMs,
      mode: r.mode,
      fellBack: r.fellBack,
    };
  },
};
```
