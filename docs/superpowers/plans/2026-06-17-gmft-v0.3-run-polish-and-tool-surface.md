### A.1 — dist runtime fix

The `dist/cli.js` artifact fails at runtime with
`ERR_MODULE_NOT_FOUND` for extensionless imports. The fix
is to switch to `moduleResolution: "NodeNext"` and add
`.js` extensions to all relative imports.

#### A.1.1 — Add `.js` extensions to all `import` and `export *` in source

For every `.ts` file in `packages/` and `apps/`, add
`.js` to relative imports (imports that start with `./`
or `../`). This is a mechanical change touching ~52
`export *` lines plus ~150 `import` lines. Use a
script + manual review.

- **Files:** all of `packages/*/src/**/*.ts` + `apps/*/src/**/*.ts`
- **Test:** all 497 existing tests must still pass

#### A.1.2 — Update `tsconfig.base.json` to NodeNext

```diff
-    "moduleResolution": "Bundler",
```

- **File:** `tsconfig.base.json`

### A.2 — SupervisorFireMarker AgentApp wiring

- **File:** `apps/gmft/src/ui/components/Transcript.tsx` (or the

- **Test:** 1 test (the marker doesn't render when there

### B.1 — 7 network recon tools

Each tool follows the existing pattern (see
`packages/tools/src/network/nmap.ts` for the template).
Each tool gets:

1. A typed module: `packages/tools/src/network/<name>.ts`
2. A Zod input/output schema
3. A `run` function that calls `runInSandbox`
4. A prereq check (binary exists in the Docker image)
5. A `Dockerfile` at `docker/<name>/Dockerfile`
6. 25 unit tests (parsing, error handling, output shape)

#### B.1.1 — `masscan_scan`
- **Image:** `gmft/masscan:0.3` (Debian slim + masscan)
- **Inputs:** `target: string, ports: string, rate: number`
- **Output:** `{ openPorts: Array<{port, protocol, service?}>, count: number }`

#### B.1.2 — `rustscan_scan`
- **Image:** `gmft/rustscan:0.3` (Debian slim + rustscan binary)
- **Inputs:** `target: string, ports: string, timeout: number`
- **Output:** same shape as masscan

#### B.1.3 — `subfinder_scan`
- **Image:** `gmft/subfinder:0.3` (Debian slim + Go binary)

#### B.1.4 — `dnsrecon_scan`

#### B.1.5 — `fierce_scan`

### C.1 — hash-chained audit.jsonl

#### C.1.1 — AuditEvent type

```typescript
type AuditEvent = {
  ts: string;  // ISO 8601
  kind: 'tool-call' | 'tool-result' | 'chokepoint-decision' |
        'session-start' | 'session-end' | 'runner-mode' | 'onboard';
  prevHash: string;  // hex SHA-256 of previous line
  hash: string;  // hex HMAC-SHA-256 of this line's canonical form
  payload: Record<string, unknown>;
};
```

- **File:** `packages/core/src/audit/types.ts`
- **Test:** `packages/core/src/audit/types.test.ts` (canonical form shape, hash recompute)

#### C.1.2 — `audit.jsonl` writer

Append-only writer. Computes the HMAC chain. Uses the
HMAC key from `~/.config/gmft/audit.key` (generated on
first event if missing, mode 0600).

- **File:** `packages/core/src/audit/writer.ts`
- **Test:** 4 tests (write first event, append, hash chain, file mode 0600)

#### C.1.3 — Auto-backup the HMAC key to the secret store

The HMAC key is added to the existing secret store
(`packages/core/src/secret/store.ts`) with the key name
`audit.hmac_key`. Auto-backup on first generation.

- **File:** `packages/core/src/audit/key.ts`,
  `packages/core/src/secret/store.ts`
- **Test:** 2 tests (key generation, key restoration from store)

#### C.1.4 — Wire the writer into the existing event emitters

`packages/core/src/agent/loop.ts`, `packages/core/src/chokepoint/`,
`packages/core/src/runner/`. Every significant event emits
an `AuditEvent`. Opt-out via `GMFT_DISABLE_AUDIT_LOG=true`.

- **Files:** 3-4 files in `packages/core/src/`
- **Test:** 2 tests (audit events emit, opt-out works)

### C.2 — `gmft audit verify`

#### C.2.1 — CLI subcommand

`gmft audit verify` walks the audit log, recomputes the
hash chain, and reports. Exit code 0 = intact, 1 = broken.

- **File:** `apps/gmft/src/cli-audit.ts` (new)
- **Test:** `apps/gmft/test/cli-audit-verify.test.ts` —
  3 tests (intact, broken, missing key)

#### C.2.2 — Output format

```
✓ audit chain intact (1247 events, 0 broken)
  last event: 2026-06-17T19:23:45.123Z (tool-result, rustscan_scan)
```

For broken:

```
✗ audit chain BROKEN at line 847
  line 847 hash mismatch
  recorded: a83f3bb3c8...
  computed: f0e6c6e9a1...
  events 848..1247 cannot be verified
```

### C.3 — `gmft audit log` and `gmft audit tail`

#### C.3.1 — `gmft audit log`

- **File:** `apps/gmft/src/cli-audit.ts` (extends C.2.1)

#### C.3.2 — `gmft audit tail`

## Risks

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| NodeNext switch breaks tests | medium | medium | Run tests early, before adding new code |
| HMAC key generation on a fresh install has a race | low | low | Single-writer pattern (the writer holds a mutex) |
| 13 new tools' Docker images can't be built in CI | medium | medium | Build images in a separate CI job; tag them with a sha |
| Audit log file grows unbounded | low | low | Add rotation in a v0.3.1 follow-up |
| `--supervisor-model` confuses users | low | low | Default is "same as primary"; opt-out is explicit |

## Open questions

See the design spec §"Open questions":
1. **Hash chain content scope** (metadata only, default).
2. **HMAC key auto-backup** (yes, default).
3. **Audit log vs session log** (parallel, not replacement).
4. **Opt-in or opt-out** (opt-out, default).

These will be resolved in the v0.3.C design review
(before C.1 starts).
