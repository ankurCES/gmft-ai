**Goal (deprecated, retained for reference):** Ship a first-run onboarding flow (provider picker → API-key prompt with live validation → model selector) backed by an extensible `ConfigField` plugin registry so future phases (tool registry, Docker sandbox, audit chain) can register their own config fields without touching the onboarding driver. Persist config to `~/.config/gmft/config.toml` (ADR-0004) and secrets to the OS keyring (or `secrets.env` fallback). Add a Models tab for in-session model switching that **auto-persists** (amends ADR-0005). Add secret redaction to the session log.

**Architecture:**

1. **Plugin registry, not hard-coded providers.** `@gmft/core` exposes `registerConfigField(field: ConfigField)` and `getConfigFields()`. The onboarding driver walks registered fields in order. Phase 1.5 registers one field (`llm-provider`) covering provider + key + model. Future phases call `registerConfigField({ id: 'tools', ... })` from their package's entry point and onboarding picks them up automatically.
2. **Provider modules are data, not behavior.** `packages/core/src/llm/providers/<id>.ts` exports a frozen `ProviderModule` object: `{ id, displayName, authFields, defaultEndpoint?, modelCatalog, validate }`. The driver iterates `PROVIDERS`; it never `import`s provider files directly (registry lookup by `id`).
3. **Keytar is best-effort, not required.** `secretStore` is an interface; two implementations: `KeytarStore` (preferred, with a load-time probe that disables itself on import failure) and `EnvFileStore` (`~/.config/gmft/secrets.env` mode 0600). The factory `createSecretStore()` returns whichever loads. The chosen backend is recorded in `config.toml` as `secrets.backend = "keytar" | "envfile"`.
4. **Validation round-trips a real provider call.** Each provider's `validate(key, endpoint?)` does the cheapest possible call: `models.list` (Anthropic/OpenAI/Google/OpenRouter) or `GET /api/tags` (Ollama). Test doubles (`FakeProvider` injected via the `providers` arg) make this unit-testable.
5. **Models tab auto-persists on switch.** Per user decision 2026-06-09, ADR-0005 §"In-TUI model selector" is amended: the model selector switches and persists atomically; no `M` key.

**Tech Stack:** TypeScript ESM, Ink v5 + React 18, keytar 7.9, smol-toml 1.3, zod 3.23, meow 13, ink-testing-library 4.

**Test budget:** 30 new `it()` cases across 9 new test files (plan called for 4; +1 file for the keytar→envfile fallback regression per user decision 2026-06-09, and provider validation grew from a single test file to two to keep the Anthropic implementation-exemplar separate from the rollup). ## Design: the `ConfigField` plugin interface

```ts
// packages/core/src/config/registry.ts
import type { GmftConfig } from './config.js';

export interface ConfigField {
  /** Stable id, dot-notation OK. e.g. 'llm-provider', 'tools.allowlist', 'sandbox.docker'. */
  id: string;
  /** Human label shown in the onboarding header. */
  label: string;
  /** Order hint, lower = earlier. Stable across versions. */
  order: number;
  /**
   * Whether the field is already satisfied given the current config. Skipped
   * when true. Allows `--provider` to skip onboarding for that field.
   */
  isConfigured(cfg: GmftConfig): boolean;
  /**
   * Render the field's interactive UI. Returns a promise that resolves with
   * a partial config patch to merge into GmftConfig, or `null` if the user
   * aborted onboarding.
   *
   * Receives a `Runtime` so the field can read/write secrets, call validators,
   * and persist to disk. The field is responsible for its own persistence:
   * return the partial; the driver merges + saves.
   */
  prompt(runtime: OnboardRuntime): Promise<Partial<GmftConfig> | null>;
}

export interface OnboardRuntime {
  /** Read a secret by logical key (e.g. 'anthropic.apiKey'). */
  getSecret(key: string): Promise<string | null>;
  /** Write a secret by logical key. */
  setSecret(key: string, value: string): Promise<void>;
  /** Validate a (provider, key, endpoint?) triple against a test double. */
  validateProvider(providerId: string, key: string, endpoint?: string): Promise<ValidateResult>;
  /** Available provider modules, in display order. */
  providers: readonly ProviderModule[];
}

const fields: ConfigField[] = [];

export function registerConfigField(field: ConfigField): void {
  if (fields.some((f) => f.id === field.id)) {
    throw new Error(`ConfigField id already registered: ${field.id}`);
  }
  fields.push(field);
}

export function getConfigFields(): readonly ConfigField[] {
  return [...fields].sort((a, b) => a.order - b.order);
}

/** Test-only. */
export function _clearConfigFields(): void {
  fields.length = 0;
}
```

The `llm-provider` field is the only registration in Phase 1.5; it is registered at the bottom of `packages/core/src/llm/providers/index.ts`:

```ts
import { registerConfigField } from '../../config/registry.js';
import { llmProviderField } from './llm-provider-field.js';
registerConfigField(llmProviderField);
```

Future phases add their own field files and import them from `@gmft/core`'s entry point (or the feature package's entry point, depending on architecture). The driver in `apps/gmft/src/cli.tsx` calls `getConfigFields()` and walks them.

// packages/core/src/config/config.ts
import { parse, stringify } from 'smol-toml';
import { readFileSync, writeFileSync, mkdirSync, existsSync } from 'node:fs';
import { join } from 'node:path';
import { homedir } from 'node:os';

export interface LlmConfig {
  provider: 'anthropic' | 'openai' | 'google' | 'openrouter' | 'ollama';
  model: string;
  endpoint?: string;
}

export interface SandboxConfig {
  mode: 'docker' | 'host';
  defaultImage?: string;
}

export interface ChokepointConfig {
  allowPrivateNetworks: boolean;
  denylist: string[];
}

export interface UiConfig {
  theme: 'auto' | 'dark' | 'light' | 'high-contrast';
}

export interface SecretsMeta {
  backend: 'keytar' | 'envfile';
}

export interface GmftConfig {
  llm: LlmConfig;
  sandbox: SandboxConfig;
  chokepoint: ChokepointConfig;
  ui: UiConfig;
  secrets: SecretsMeta;
  [k: string]: unknown; // forward-compat: preserve unknown keys
}

export function defaultConfig(): GmftConfig {
  return {
    llm: { provider: 'anthropic', model: 'claude-3-5-haiku-latest' },
    sandbox: { mode: 'host' },
    chokepoint: { allowPrivateNetworks: false, denylist: [] },
    ui: { theme: 'auto' },
    secrets: { backend: 'envfile' },
  };
}

export function configDir(): string {
  return process.env.XDG_CONFIG_HOME ?? join(homedir(), '.config');
}

export function configPath(): string {
  return join(configDir(), 'gmft', 'config.toml');
}

export function loadConfig(): GmftConfig {
  const p = configPath();
  if (!existsSync(p)) return defaultConfig();
  const text = readFileSync(p, 'utf8');
  const raw = parse(text) as Record<string, unknown>;
  return { ...defaultConfig(), ...raw } as GmftConfig;
}

export function saveConfig(cfg: GmftConfig): void {
  const p = configPath();
  mkdirSync(join(configDir(), 'gmft'), { recursive: true });
  writeFileSync(p, stringify(cfg as Record<string, unknown>));
}
```

// packages/core/src/config/secrets.ts
import { readFileSync, writeFileSync, mkdirSync, existsSync, statSync, chmodSync } from 'node:fs';
import { join } from 'node:path';
import { homedir } from 'node:os';

export type SecretBackend = 'keytar' | 'envfile';

export interface SecretStore {
  readonly backend: SecretBackend;
  get(key: string): Promise<string | null>;
  set(key: string, value: string): Promise<void>;
  delete(key: string): Promise<void>;
}

export function secretsEnvDir(): string {
  return process.env.XDG_CONFIG_HOME ?? join(homedir(), '.config');
}

export function envPath(): string {
  return join(secretsEnvDir(), 'gmft', 'secrets.env');
}

class EnvFileStore implements SecretStore {
  readonly backend: SecretBackend = 'envfile';
  constructor(private readonly service: string) {}

  private async readAll(): Promise<Record<string, string>> {
    const p = envPath();
    if (!existsSync(p)) return {};
    const text = readFileSync(p, 'utf8');
    const out: Record<string, string> = {};
    for (const line of text.split('\n')) {
      const m = line.match(/^([A-Z0-9_]+)\s*=\s*(.*)$/);
      if (m) out[m[1]!] = m[2] ?? '';
    }
    return out;
  }

  private async writeAll(map: Record<string, string>): Promise<void> {
    const p = envPath();
    mkdirSync(join(secretsEnvDir(), 'gmft'), { recursive: true });
    const body = Object.entries(map)
      .map(([k, v]) => `${k}=${v.replace(/\n/g, '\\n')}`)
      .join('\n');
    writeFileSync(p, body + '\n');
    chmodSync(p, 0o600);
  }

  private compositeKey(key: string): string {
    return `${this.service.toUpperCase().replace(/[^A-Z0-9]/g, '_')}_${key.replace(/\./g, '_').toUpperCase()}`;
  }

  async get(key: string): Promise<string | null> {
    const all = await this.readAll();
    return all[this.compositeKey(key)] ?? null;
  }

  async set(key: string, value: string): Promise<void> {
    const all = await this.readAll();
    all[this.compositeKey(key)] = value;
    await this.writeAll(all);
  }

  async delete(key: string): Promise<void> {
    const all = await this.readAll();
    delete all[this.compositeKey(key)];
    await this.writeAll(all);
  }
}

class KeytarStore implements SecretStore {
  readonly backend: SecretBackend = 'keytar';
  // Lazily imported; if the require throws, the caller falls back to EnvFileStore.
  private mod: typeof import('keytar') | null = null;
  constructor(private readonly service: string) {}

  private async load(): Promise<typeof import('keytar')> {
    if (this.mod) return this.mod;
    this.mod = await import('keytar');
    return this.mod;
  }

  async get(key: string): Promise<string | null> {
    const k = await this.load();
    return k.getPassword(this.service, key);
  }
  async set(key: string, value: string): Promise<void> {
    const k = await this.load();
    await k.setPassword(this.service, key, value);
  }
  async delete(key: string): Promise<void> {
    const k = await this.load();
    await k.deletePassword(this.service, key);
  }
}

export interface CreateOpts {
  service: string;
}

/**
 * Returns the best SecretStore available. If `keytar` fails to load (native
 * binding missing — typical on a CI box without libsecret), falls back to
 * `EnvFileStore` and records the choice in the caller's `config.toml` via
 * `backend`.
 */
export async function createSecretStore(opts: CreateOpts): Promise<SecretStore> {
  const probe = new KeytarStore(opts.service);
  try {
    // get() does a lazy import; that import is the failure point on broken builds.
    // Probe with a known-missing key: if getPassword returns null we're fine.
    await probe.get('__gmft_probe__');
    return probe;
  } catch {
    return new EnvFileStore(opts.service);
  }
}

```ts
// packages/core/src/llm/onboard.ts
import type { GmftConfig, LlmConfig } from '../config/config.js';
import { defaultConfig } from '../config/config.js';
import type { ConfigField, OnboardRuntime } from '../config/registry.js';
import type { ProviderModule } from './providers/types.js';

export interface RunOnboardingOpts {
  fields: readonly ConfigField[];
  providers: readonly ProviderModule[];
  /** Builds the runtime handed to each field's prompt(). Lets the caller inject fakes. */
  runtimeFactory: () => Omit<OnboardRuntime, 'providers'> & { providers: readonly ProviderModule[] };
  save: (cfg: GmftConfig) => void;
}

export type OnboardResult =
  | { ok: true; config: GmftConfig }
  | { ok: false; reason: 'aborted' | 'invalid_state' };

/**
 * Pure-ish driver. Walks the registered ConfigFields in order, calls each
 * one's prompt() with a runtime, merges the returned partial into the
 * current config, persists via `save()`, and returns the final config.
 *
 * Does NOT render any UI. The actual ink components live in apps/gmft; this
 * driver is what the App calls inside the onboarding flow.
 */
export async function runOnboarding(opts: RunOnboardingOpts): Promise<OnboardResult> {
  let cfg: GmftConfig = defaultConfig();
  const runtime = opts.runtimeFactory();

  for (const field of opts.fields) {
    if (field.isConfigured(cfg)) continue;
    const patch = await field.prompt(runtime);
    if (patch === null) return { ok: false, reason: 'aborted' };
    cfg = mergeConfig(cfg, patch);
  }

  // Record the secrets backend the runtime is using, if it exposes one.
  if ((runtime as { backend?: 'keytar' | 'envfile' }).backend) {
    cfg.secrets = { backend: (runtime as { backend: 'keytar' | 'envfile' }).backend };
  }

  opts.save(cfg);
  return { ok: true, config: cfg };
}

function mergeConfig(base: GmftConfig, patch: Partial<GmftConfig>): GmftConfig {
  const out: GmftConfig = { ...base, ...patch };
  if (patch.llm) out.llm = { ...base.llm, ...patch.llm } as LlmConfig;
  return out;
}
```

  useInput((input, key) => {
    if (key.upArrow) setIdx((i) => (i - 1 + providers.length) % providers.length);

    if (key.return) onPick(providers[idx]!.id);
  });

  label: string;
  description?: string;

  currentModel: string;

   * the parent persists to config.toml atomically; there is no separate
   * "press M to persist" step.
   */
  onChange: (modelId: string) => void;

export function ModelsTab({ provider, currentModel, models, onChange }: ModelsTabProps): React.JSX.Element {

    if (key.downArrow) setIdx((i) => (i + 1) % models.length);
    if (key.return && models[idx]) onChange(models[idx]!.id);
  });
  return (
    <Box flexDirection="column">
      <Text>Provider: {provider}</Text>
      <Text>Current model: {currentModel}</Text>

- [ ] **Step 3: Create the llm-provider-field module**

To avoid a circular import (the field uses ink components, which live in
`apps/gmft`; the field is registered into `@gmft/core`'s registry), the
field is split into two files:

1. **`packages/core/src/llm/llm-provider-field.ts`** — the state machine.
   Exports `llmProviderField: ConfigField` whose `prompt()` walks a list of
   "steps" and returns a partial config. The step list is data, not UI:
   ```ts
   export type ProviderStep =
     | { kind: 'pickProvider' }
     | { kind: 'enterKey'; fieldId: string; label: string }
     | { kind: 'pickModel' };
   export interface ProviderStepsResult {
     steps: ProviderStep[];
     /** Resolve a step's UI prompt. Injected by apps/gmft. */
     resolveStep(step: ProviderStep, rt: OnboardRuntime): Promise<string | null>;
   }
   ```
2. **`apps/gmft/src/onboard/bind-provider-ui.tsx`** — the UI binder. Exports
   `bindProviderUI(): ProviderStepsResult` that returns the resolver, which
   uses ink's `render()` to mount the actual `ProviderPicker` / `ApiKeyPrompt`
   / `ModelSelector` and resolves the promise via their `onPick` / `onSubmit`.

The cli in Step 2 calls `bindProviderUI()` to get a resolver, then passes
the resolver into the field via a new optional `ConfigField.bindUI` hook:

```ts
export interface ConfigField {
  // ...existing
  /** Optional: replaces the default no-UI prompt flow with a UI-driven one. */
  bindUI?(rt: OnboardRuntime): ProviderStepsResult;
}
```

If `bindUI` is present, `runOnboarding` calls it and uses the resolver; if
absent, `runOnboarding` falls back to a CLI flow (out of scope for v0.1).

**File 1: `packages/core/src/llm/llm-provider-field.ts`**

```ts
import type { ConfigField, OnboardRuntime } from '../config/registry.js';
import type { GmftConfig } from '../config/config.js';

export type ProviderStep =
  | { kind: 'pickProvider' }
  | { kind: 'enterKey'; fieldId: string; label: string }
  | { kind: 'pickModel' };

export interface ProviderStepsResult {
  steps: ProviderStep[];
  resolveStep(step: ProviderStep, rt: OnboardRuntime): Promise<string | null>;
}

export const llmProviderField: ConfigField = {
  id: 'llm-provider',
  label: 'LLM provider',
  order: 10,
  isConfigured: (cfg) => cfg.llm.provider !== undefined && cfg.llm.model !== '',
  async prompt(rt: OnboardRuntime) {
    const ui = this.bindUI?.(rt);
    if (!ui) {
      throw new Error(
        'llm-provider field requires bindUI; apps/gmft must call bindProviderUI()',
      );
    }
    const stepValues: Record<string, string> = {};
    for (const step of ui.steps) {
      const v = await ui.resolveStep(step, rt);
      if (v === null) return null;
      stepValues[step.kind] = v;
    }
    const providerId = stepValues['pickProvider']!;
    const provider = rt.providers.find((p) => p.id === providerId);
    if (!provider) return null;
    const key = stepValues['enterKey'];
    if (key) {
      await rt.setSecret(`${provider.id}.${provider.authFields[0]!.id}`, key);
      const v = await rt.validateProvider(provider.id, key, provider.defaultEndpoint);
      if (!v.ok) throw new Error(`Validation failed: ${v.reason}`);
    } else if (provider.defaultEndpoint) {
      const v = await rt.validateProvider(provider.id, '', provider.defaultEndpoint);
      if (!v.ok) throw new Error(`Cannot reach ${provider.defaultEndpoint}: ${v.reason}`);
    }
    const modelId = stepValues['pickModel']!;
    return {
      llm: {
        provider: provider.id,
        model: modelId,
        ...(provider.defaultEndpoint ? { endpoint: provider.defaultEndpoint } : {}),
      },
    } satisfies Partial<GmftConfig>;
  },
  bindUI(_rt: OnboardRuntime): ProviderStepsResult {
    throw new Error('bindUI not set; apps/gmft must override via bindProviderUI()');
  },
};
```

  ProviderStep,
  ProviderStepsResult,
} from '@gmft/core/llm/llm-provider-field';

import { ApiKeyPrompt } from '../ui/components/ApiKeyPrompt.js';
import { ModelSelector } from '../ui/components/ModelSelector.js';

export function bindProviderUI(rt: OnboundRuntime): ProviderStepsResult {
  return {
    steps: [
      { kind: 'pickProvider' },
      // The 'enterKey' step is added by the field at runtime if needed; the
      // binder resolves it generically by looking at the picked provider.
      { kind: 'pickModel' },
    ],
    async resolveStep(step, rt2) {
      if (step.kind === 'pickProvider') {
        return new Promise((resolve) => {
          const { unmount } = inkRender(
            React.createElement(ProviderPicker, {
              providers: rt2.providers,
              onPick: (id) => { unmount(); resolve(id); },
            }) as React.ReactElement,
          );
        });

      if (step.kind === 'pickModel') {

        // the field needs. The binder doesn't know the provider; defer to
        // the field's caller. We resolve with a sentinel that the field
        // recognizes — but cleaner: have the field call resolveStep itself
        // for the model step (since it knows the provider). See Step 3a.
        throw new Error('pickModel resolution lives in the field, not the binder');
      }
      return null;
    },
  };
}

// packages/core/src/session/log.ts
import { writeFile, readFile, appendFile, mkdir } from 'node:fs/promises';
import { dirname } from 'node:path';

const REDACT_RE = /(api[_-]?key|token|secret)\s*[=:]\s*\S+/gi;

/**
 * Replace any (api_key|token|secret)\s*[=:]\s*<value> with `[REDACTED]`.
 * The regex is case-insensitive and tolerant of `=`, `:`, and surrounding
 * whitespace. It matches the variable name up to a non-word boundary and
 * captures the trailing value as the longest run of non-whitespace.
 */
export function redact(line: string): string {
  return line.replace(REDACT_RE, (_m, head: string) => `${head}=[REDACTED]`);
}

export interface Turn {
  ts?: number;
  role: 'user' | 'assistant' | 'system' | 'tool';
  content: string;
  [k: string]: unknown;
}

/** Write a single turn as a JSONL line. The content is redacted before write. */
export async function writeLine(path: string, turn: Turn): Promise<void> {
  await mkdir(dirname(path), { recursive: true });
  const ts = turn.ts ?? Date.now();
  const content = redact(turn.content);
  const safe: Turn = { ...turn, ts, content };
  await appendFile(path, JSON.stringify(safe) + '\n', 'utf8');
}

export async function readLog(path: string): Promise<string[]> {
  try {
    const text = await readFile(path, 'utf8');
    return text.split('\n').filter((l) => l.length > 0);
  } catch {
    return [];
  }
}

The plan documents the fallback (move the field state machine to `@gmft/core` and have `apps/gmft` bind the UI).
