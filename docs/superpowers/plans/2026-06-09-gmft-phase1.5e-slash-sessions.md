# Phase 1.5e — Slash commands + session persistence + resume-on-start

> Slice of v0.1 Phase 2 (`/help`, `/clear`, `/model <id>`, `/provider <id>`, `/exit`, `/session new|list/load <id>`, plus session persistence + resume-on-start) that lands before Phase 3 (chokepoint) because it touches the App/CLI seam that 1.5d also touches.

## What's already in 1.5d

- `packages/core/src/session/log.ts` exports `Turn`, `appendTurn`, `readLog`, `redactSecrets` — the JSONL primitives 1.5e will use (shipped in 1.5b).
- `apps/gmft/src/ui/hooks/useAgent.ts` is the testable streaming hook. **It is currently unused by `AgentApp.tsx`** — `AgentApp` rolls its own streaming loop. For 1.5e we will refactor `AgentApp` to compose `useAgent` so the persistence + slash-command wiring can subscribe to one canonical history source.
- `apps/gmft/src/App.tsx` already has stub `/help` and `/clear` branches in `handleSubmit`. They are hardcoded into the App, not the chat layer.
- `packages/core/src/config/config.ts` exposes `configDir()` (XDG-aware). We will add `sessionDir()` and `sessionPath(id)` next to it.

## Architecture (the seam 1.5e introduces)

```
cli.tsx
  ├── loadConfig() + onboarding (existing)
  ├── resume check: if last session exists + not --no-resume → "Resume <id>? [Y/n]" (NEW)
  ├── AgentApp (refactored to compose useAgent)
  └── on exit → appendTurn() the final user+assistant pair if a turn ran

AgentApp (refactored)
  ├── useAgent({ runTurn, system, initialHistory })  ← single source of truth
  ├── useSlashCommands()                              ← consumes history + submit
  └── <App onSubmit={handleAnything}>

useSlashCommands (NEW, in apps/gmft/src/session/commands.ts)
  ├── Returns { handleSubmit(text) } that:
  │   ├── If text starts with `/` → dispatch to command, do NOT call runTurn
  │   │   ├── /help      → push help message
  │   │   ├── /clear     → clear messages + log
  │   │   ├── /model X   → switch model in-memory (no LLM call)
  │   │   ├── /provider X→ switch provider + model in-memory
  │   │   ├── /exit      → onExit() + exit()
  │   │   ├── /session new       → start new session
  │   │   ├── /session list      → show session ids
  │   │   └── /session load <id> → load + replay
  │   └── else → call runTurn via useAgent
  └── Side effects: persist user+assistant to session log on every turn

SessionStore (NEW, in apps/gmft/src/session/store.ts)
  ├── sessionDir() → `${configDir()}/gmft/sessions/`
  ├── currentSessionPath() → sessionDir() + 'current.jsonl'
  ├── sessionPath(id) → sessionDir() + `${id}.jsonl`
  ├── listSessions() → [{id, mtime, turns}]
  ├── startSession(id?) → id (default: ISO timestamp slug)
  ├── appendTurn(turn) → fs.appendFile
  ├── loadSession(id) → Turn[]
  └── clear() → fs.unlink current.jsonl
```

## Why refactor AgentApp onto useAgent now

- `useAgent` already owns `history` + `streaming` + `error` state.
- `AgentApp`'s bespoke loop duplicates that and lives outside the seam.
- 1.5e needs *one* history source that both slash commands and the persistence layer can read. Without the refactor, slash commands would either have to (a) go through `App.handleSubmit` and live in `App.tsx` (where they already are as stubs) or (b) reach into `AgentApp` to mirror state. Both are worse than the refactor.
- The refactor itself is 1 commit and does not change any test that already passes.

8. `apps/gmft/src/App.tsx` — remove the stub `/help` and `/clear` handlers (now in commands.ts). Pass `onSubmit` straight through. ## Design decisions called out (push back if you disagree)

1. **JSONL location is `~/.config/gmft/sessions/<id>.jsonl`** (XDG-aware via existing `configDir()`). The `current` pointer is a separate file at `current-session-id` containing just the id. No new env var.
2. **Sessions are immutable once created.** `/session new` always picks a fresh id (default `YYYYMMDD-HHmmss-<6hex>`). `/session load` switches the active session but does not delete the old one. Phase 1.5e is read/write only — no rename, no delete. Use `rm` from the shell if you want to clean up.
3. **/model and /provider are in-memory only.** No config rewrite. Switching persists for the current session; exit-without-Ctrl-C (i.e. SIGKILL) loses the switch. Same UX as the v0.1 plan §2.10 said. Persistence-of-switch lands in 1.5f if we want it.
4. **No `last-active` rotation.** Resume prompt is binary: resume the current session or start a new one. If you want to keep multiple sessions in rotation, the id swap is the seam — a 1.5e+1 task.
5. **No token counter yet.** Each turn records `ts` and `role` and `content`. Tokens land with the chokepoint in Phase 3 (which actually has a tokenizer dependency).
6. **/session list is in-memory + filesystem scan** (not part of useAgent history). Cheap to add; useful for the UX.
7. **The refactor onto `useAgent` is in scope.** Without it the slash-command logic would be split between App and commands.ts, which is the failure mode we are trying to prevent.

## Test plan

- `packages/core/test/session-paths.test.ts` (3-4 tests): `sessionDir()` honors XDG; `sessionPath(id)` joins correctly; missing dir does not throw (the consumer is responsible for `mkdir -p`).
- `apps/gmft/test/session-store.test.ts` (5-6 tests): write 3 turns to a tmp dir, read back 3, list returns the right id + mtime, clear() removes the file, start() with explicit id is honored.
- `apps/gmft/test/slash-commands.test.ts` (8-10 tests): table-driven, one per command. `/help` returns the help text; `/clear` returns `{kind:'handled'}` and the dispatcher leaves the existing messages alone; `/model foo` returns `{kind:'handled'}` and sets the model; `/exit` returns `{kind:'exited'}`; `/session new` clears the log; `/session load bogus` returns an error message and does not crash.
- `apps/gmft/test/agent-app.test.tsx` (2-3 tests): refactor regression — `AgentApp` renders `<App>`, streaming a turn hydrates the chat, and `onTurnComplete` fires with the final user+assistant pair. Reuses the test from 1.5d's `useAgent.test.tsx` pattern.
- `apps/gmft/test/resume-prompt.test.tsx` (3 tests): renders the id + turn count, `Y` calls `onResume` with the id, `n` calls `onStartNew`.
- `apps/gmft/test/cli-resume.test.tsx` (2-3 tests): cli spawns AgentApp with `initialHistory` when the session has turns; cli spawns with empty history when `--no-resume` or no session exists.

**Test budget**: 5+5+9+3+3+3 = ~28 new. Total: 97 → ~125.

## What ships

- 10 slash commands: `/help`, `/clear`, `/model <id>`, `/provider <id>`, `/exit`, `/session new|list|load <id>`, and a `--resume <id>` / `--no-resume` CLI flag.
- JSONL session log per session under `~/.config/gmft/sessions/`.
- A "current" pointer file.
- Resume prompt on next launch: "Resume `<id>` (N turns)? [Y/n]".
- ~28 new tests; 0 regressions in the existing 97.
- One CHANGELOG entry + tag `v0.1.0-phase1.5e`.

## What does NOT ship

- Per-tool-call log entries (no chokepoint yet).
- Token counts.
- /model and /provider persistence to config.
- /session rename or delete.
- Multi-session resume picker (binary only).
- Backups / rotation.
